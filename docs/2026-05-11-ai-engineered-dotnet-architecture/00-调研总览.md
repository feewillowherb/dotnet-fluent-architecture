# AI 工程化 .NET 架构调研总览

**日期**: 2026-05-11
**来源**: MaterialClient 项目代码分析
**目标**: 提取代码风格与 AI 工程化格式，沉淀为通用参考

## 调研范围

从 MaterialClient 项目中提取以下通用模式：

1. **AI 协作配置** — CLAUDE.md、AGENTS.md、OpenSpec
2. **项目构建体系** — Directory.Build.props、Directory.Packages.props
3. **代码风格规范** — C# 13 现代语法、命名约定、文件组织
4. **架构模式** — ABP 框架集成、DDD、分层架构
5. **服务注册模式** — ABP 隐式注册 + AutoConstructor
6. **数据访问模式** — EF Core + ABP 仓储
7. **HTTP 客户端模式** — Refit + Polly
8. **事件通信模式** — ReactiveUI MessageBus
9. **测试策略** — TDD + ABP 集成测试
10. **治理体系** — Constitution、Error Cases

## 关键发现

1. MaterialClient 有一套成熟的 AI 工程化协作体系（CLAUDE.md + AGENTS.md + OpenSpec），可以将通用部分提取为模板
2. 代码风格规范中有大量 NON-NEGOTIABLE 约束，这些约束通过 AGENTS.md 传递给 AI Agent，确保代码一致性
3. ABP 框架 + AutoConstructor 的 DI 模式极大减少了样板代码
4. OpenSpec 工作流提供规范驱动的变更管理，适合 AI 辅助开发场景
5. OpenSpec 工作流提供规范驱动的变更管理，constitution.md 作为项目宪章定义核心原则

## 风险与限制

- Sample 项目中的部分代码（如 EF Core 迁移）需要实际运行 `dotnet ef` 命令后才能完整
- ABP 框架版本绑定 .NET 10.0，不适用于更低版本
- 未包含 Avalonia UI 相关的 MVVM 代码（可在后续迭代中补充）
