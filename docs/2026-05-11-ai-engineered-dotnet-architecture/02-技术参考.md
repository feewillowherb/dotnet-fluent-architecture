# 技术参考

## 1. 构建体系

### Directory.Build.props

所有项目自动继承的构建属性：

```xml
<PropertyGroup>
    <TargetFramework>net10.0</TargetFramework>
    <Nullable>enable</Nullable>
    <ImplicitUsings>enable</ImplicitUsings>
    <RootNamespace>FluentSample</RootNamespace>
</PropertyGroup>
<ItemGroup>
    <PackageReference Include="AutoConstructor">...</PackageReference>
</ItemGroup>
```

**关键点**：
- `net10.0` 确保所有项目使用统一目标框架
- `Nullable` 和 `ImplicitUsings` 强制启用现代 C# 特性
- `AutoConstructor` 源生成器对所有项目可用

### Directory.Packages.props

Central Package Management (CPM)，所有包版本集中管理：

```xml
<PropertyGroup>
    <ManagePackageVersionsCentrally>true</ManagePackageVersionsCentrally>
</PropertyGroup>
<ItemGroup>
    <PackageVersion Include="AutoConstructor" Version="5.6.0" />
    <PackageVersion Include="Volo.Abp.Core" Version="10.0.1" />
    <!-- ... -->
</ItemGroup>
```

**关键点**：
- 项目引用包时不带版本号，版本统一在 CPM 文件中定义
- 确保跨项目版本一致性

## 2. ABP 框架集成

### 模块依赖链

```
App Module → Core Module → AbpEntityFrameworkCoreModule → AbpAutofacModule
```

### 服务注册模式

**推荐方式**：ABP 隐式注册 + `[AutoConstructor]`

```csharp
[AutoConstructor]
public partial class MyService : IMyService, ITransientDependency
{
    private readonly IRepository<MyEntity, Guid> _repository;
    private readonly ILogger<MyService> _logger;
    // AutoConstructor 自动生成构造函数
}
```

**ABP 生命周期接口**：
- `ITransientDependency` — 每次请求新建实例
- `IScopedDependency` — 每个 scope 一个实例
- `ISingletonDependency` — 全局单例

### ABP 模块配置

```csharp
[DependsOn(
    typeof(AbpEntityFrameworkCoreModule),
    typeof(AbpEntityFrameworkCoreSqliteModule)
)]
public class CoreModule : AbpModule
{
    public override void ConfigureServices(ServiceConfigurationContext context)
    {
        // 注册 DbContext + 默认仓储
        context.Services.AddAbpDbContext<MyDbContext>(options =>
        {
            options.AddDefaultRepositories(true);
        });
    }
}
```

## 3. DDD 实体模式

### 实体基类选择

| 场景 | 基类 |
|------|------|
| 普通实体 | `Entity<TKey>` |
| 需要审计字段 | `FullAuditedEntity<TKey>` |
| 聚合根 | `FullAuditedAggregateRoot<TKey>` |

### 富模型设计（Rich Domain Model）

```csharp
public class SampleItem : Entity<Guid>
{
    public string Name { get; private set; }  // private set 确保封装性

    public SampleItem(string name, ...)       // 业务构造函数
    {
        Name = GuardName(name);               // 验证逻辑
    }

    private SampleItem() { }                  // EF Core 构造函数

    public decimal CalculateTotalPrice() => Quantity * UnitPrice;  // 领域行为

    public void UpdateQuantity(int newQuantity)  // 命令方法
    {
        Quantity = GuardQuantity(newQuantity);
    }
}
```

### Record 替代 Tuple

```csharp
// PROHIBITED
(string Name, int Age) GetUserInfo() => ("John", 30);

// REQUIRED
record UserInfo(string Name, int Age);
UserInfo GetUserInfo() => new UserInfo("John", 30);
```

## 4. 数据访问模式

### ABP 仓储模式

```csharp
// 在服务中使用仓储（ABP 自动注入）
public class MyService : DomainService, ITransientDependency
{
    private readonly IRepository<SampleItem, Guid> _repository;

    public async Task<SampleItem> CreateAsync(string name)
    {
        var item = new SampleItem(name, 1, 10m);
        return await _repository.InsertAsync(item, autoSave: true);
    }
}
```

### EF Core 配置

```csharp
public class MyDbContext : AbpDbContext<MyDbContext>
{
    public DbSet<SampleItem> Items => Set<SampleItem>();

    protected override void OnModelCreating(ModelBuilder modelBuilder)
    {
        base.OnModelCreating(modelBuilder);
        modelBuilder.Entity<SampleItem>(b =>
        {
            b.ToTable("SampleItems");
            b.HasKey(x => x.Id);
            // Fluent API 配置...
        });
    }
}
```

## 6. 事件通信模式

public class StatusChangedMessage(Guid ItemId, ItemStatus NewStatus) { }

## 8. 测试策略

### 测试约定

- **框架**: xUnit + Shouldly + NSubstitute
- **TDD**: 测试先行 → 用户批准 → 测试失败 → 实现
- **测试 DTO 命名**: `TestDto` 后缀
- **InternalsVisibleTo**: 测试项目可访问 internal 成员

### 测试代码示例

```csharp
public class SampleItemTests
{
    [Fact]
    public void Constructor_Should_Set_Properties_Correctly()
    {
        var item = new SampleItem("Test", 10, 25.50m);
        item.Name.ShouldBe("Test");
        item.Quantity.ShouldBe(10);
    }

    [Fact]
    public void TransitionTo_Should_Throw_For_Invalid_Transitions()
    {
        var item = new SampleItem("Test", 1, 1m);
        Should.Throw<InvalidOperationException>(
            () => item.TransitionTo(ItemStatus.Completed));
    }
}
```
