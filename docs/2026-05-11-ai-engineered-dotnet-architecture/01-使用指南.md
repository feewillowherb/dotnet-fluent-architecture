# 使用指南

## 快速开始：基于 FluentSample 创建新项目

### 1. 复制模板

```bash
cp -r repos/FluentSample /path/to/your-project
cd /path/to/your-project
```

### 2. 全局替换命名空间

将所有 `FluentSample` 替换为你的项目名称：

```bash
# 在所有文件中替换命名空间
find . -type f \( -name "*.cs" -o -name "*.csproj" -o -name "*.md" -o -name "*.json" -o -name "*.sln" \) \
  -exec sed -i 's/FluentSample/YourProject/g' {} +
```

### 3. 初始化 Git

```bash
git init
git add .
git commit -m "Initial project from FluentSample template"
```

### 4. 生成 EF Core 迁移

```bash
cd src/YourProject.Core
dotnet ef migrations add InitialCreate --startup-project ../YourProject.App
```

### 5. 运行测试

```bash
cd tests/YourProject.Core.Tests
dotnet test
```

## 项目结构说明

```
your-project/
├── CLAUDE.md                    # AI 协作入口（Claude Code 读取）
├── AGENTS.md                    # Agent 行为准则和编码规范
├── Directory.Build.props        # 统一构建属性
├── Directory.Packages.props     # NuGet 包版本管理
├── openspec/                    # 规范驱动开发
│   ├── project.md               # 项目约定和技术栈
│   ├── specs/                   # 功能规范（当前能力的真相源）
│   ├── changes/                 # 待实施变更
│   ├── docs/                    # 技术文档
│   └── archive/                 # 已完成的变更
├── docs/
├── src/
│   ├── YourProject.App/         # 应用入口
│   │   ├── Program.cs           # 启动代码
│   │   ├── YourProjectAppModule.cs  # ABP 模块
│   │   └── appsettings.json     # 配置文件
│   └── YourProject.Core/        # 核心业务逻辑
│       ├── YourProjectCoreModule.cs  # 核心 ABP 模块
│       ├── Api/                 # Refit 接口和 DTO
│       ├── Configuration/       # 配置类
│       ├── Entities/            # 领域实体
│       ├── EntityFrameworkCore/ # 数据库上下文
│       ├── Events/              # MessageBus 消息
│       ├── Services/            # 业务服务
│       ├── Utils/               # 静态工厂方法
│       └── Providers/           # DI 工厂服务
├── tests/
│   └── YourProject.Core.Tests/  # 测试项目
└── YourProject.sln              # 解决方案文件
```

## AI 工程化配置说明

### CLAUDE.md

Claude Code 的入口文件。内容极简，指向 AGENTS.md：

```
请阅读 AGENTS.md 中的项目约定与 OpenSpec 参考。
```

### AGENTS.md

核心 AI 协作文件，包含：
- OpenSpec 工作流说明
- 服务注册约定
- 编码规范（NON-NEGOTIABLE 标记的强制规则）
- 设计模式原则
- 治理规则

### openspec/ 目录

规范驱动开发工作流目录结构：
- `specs/` — 功能规范（当前能力的真相源）
- `changes/` — 待实施变更
- `docs/` — 技术文档
- `archive/` — 已完成的变更
