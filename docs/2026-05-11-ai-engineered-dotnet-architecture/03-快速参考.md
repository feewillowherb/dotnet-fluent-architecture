# AI 工程化配置速查

## 文件清单

创建新项目时需要配置的 AI 工程化文件：

| 文件 | 作用 | 优先级 |
|------|------|--------|
| `CLAUDE.md` | Claude Code 入口，指向 AGENTS.md | 必须 |
| `AGENTS.md` | Agent 行为准则、编码规范、NON-NEGOTIABLE 约束 | 必须 |
| `openspec/project.md` | 项目约定、技术栈、架构原则 | 推荐 |
| `openspec/` | 规范驱动开发（specs/changes/archive） | 推荐 |

## CLAUDE.md 模板

```markdown
请阅读 `AGENTS.md` 中的项目约定与 OpenSpec 参考。Agent 可按用户要求直接修改代码，也可选择通过 OpenSpec 工作流进行变更。
```

## Directory.Build.props 模板

```xml
<Project>
  <PropertyGroup>
    <TargetFramework>net10.0</TargetFramework>
    <Nullable>enable</Nullable>
    <ImplicitUsings>enable</ImplicitUsings>
    <RootNamespace>YourProject</RootNamespace>
  </PropertyGroup>
  <ItemGroup>
    <PackageReference Include="AutoConstructor">
      <PrivateAssets>all</PrivateAssets>
      <IncludeAssets>runtime; build; native; contentfiles; analyzers</IncludeAssets>
    </PackageReference>
  </ItemGroup>
</Project>
```

## 常用 NON-NEGOTIABLE 规则速查

| 规则 | 说明 |
|------|------|
| File-scoped namespaces | `namespace X;` 而非 `{ }` |
| Primary constructors | 简单类使用主构造函数 |
| Collection expressions | `[]` 替代 `new List<T>()` |
| Record over tuple | 禁止 `(string, int)` 作为返回值 |
| AutoConstructor | DI 构造函数使用源生成器 |
| English identifiers | 变量名和字段禁止非英文字符 |
| Interface + Impl same file | < 1000 行时放同一文件，Interface 在前 |
| Single Source of Truth | 常量和默认值定义在唯一位置 |
| Plans in English | 实施计划必须用英文撰写 |

## 服务注册模式速查

```csharp
// 接口 + 实现（同一文件）
public interface IMyService { Task DoWorkAsync(); }

[AutoConstructor]
public partial class MyService : DomainService, IMyService, ITransientDependency
{
    private readonly IRepository<MyEntity, Guid> _repository;
    // AutoConstructor 自动生成构造函数注入
}
```

## 消息定义模式速查

```csharp
// Events/ 目录下，使用 primary constructor
public class MyEventMessage(Guid EntityId, SomeStatus NewStatus)
{
    public Guid EntityId { get; } = EntityId;
    public SomeStatus NewStatus { get; } = NewStatus;
}
```
